# Hearloop: Code Patterns & Conventions

**Reference this when writing code to match project style.**

## TypeScript & Import Rules

### Import Order (All Files)
```typescript
// 1. External packages
import Fastify from 'fastify';
import { Job } from 'bullmq';

// 2. Relative imports (./)
import { db } from '../lib/db';
import { transcribe } from './transcribe';

// 3. Type imports (separate)
import type { Session, Recording } from '../types';
```

### File Naming
- `camelCase.ts` for most files
- `PascalCase.tsx` for React components
- Job files: `verb-noun.ts` (e.g., `deliver-webhook.ts`)
- Route files: `noun.ts` (e.g., `sessions.ts`)

## Backend Patterns (`api/`)

### Fastify Route Registration
```typescript
// routes/my-route.ts
export async function setupMyRoute(app: FastifyInstance) {
  app.post('/api/my-endpoint', async (request, reply) => {
    // Validate input
    // Call lib functions
    // Return JSON
    return { data: result };
  });
}

// index.ts - Register it
await setupMyRoute(app);
```

### Job Handlers
```typescript
// jobs/my-job.ts
export async function myJobHandler(job: Job) {
  console.log(`Processing ${job.id}`);
  
  // Do work
  const result = await someAsyncWork();
  
  // Return result (optional)
  return result;
  
  // Throw to retry (BullMQ handles retries)
  // throw new Error('retry me');
}

// lib/queue.ts - Register it
queue.process('my-job', myJobHandler);
```

### Database Queries (Kysely)
```typescript
import { db } from './db';

// Insert
await db.insertInto('sessions')
  .values({ id: uuid(), partnerId, status: 'created' })
  .execute();

// Select
const session = await db.selectFrom('sessions')
  .selectAll()
  .where('id', '=', sessionId)
  .executeTakeFirstOrThrow();

// Update
await db.updateTable('sessions')
  .set({ status: 'completed' })
  .where('id', '=', sessionId)
  .execute();
```

### Widget Vanilla JS Pattern
```javascript
// components/widget.js
(function() {
  'use strict';
  
  class HearloopWidget {
    constructor(config) {
      this.config = config;
      this.sessionToken = config.sessionToken;
    }
    
    async record() {
      const audioData = await this.captureAudio();
      await this.submitRecording(audioData);
    }
  }
  
  window.HearloopWidget = HearloopWidget;
})();
```

## Error Handling

### API Errors
```typescript
// Return standard error format
reply.code(400).send({
  error: 'invalid_input',
  message: 'Session token is invalid'
});
```

### Job Errors
```typescript
// Throw to trigger retry
if (!data) {
  throw new Error('Data not found - will retry');
}

// Or fail job
throw new Error('Fatal error - no retry');
```


### Frontend (.env.local)
```
NEXT_PUBLIC_API_URL=http://localhost:3000 (dev) or https://api.hearloop.com (prod)
```

## No-Nos
- ❌ `any` types (use `unknown` with guards)
- ❌ Callbacks (always async/await)
- ❌ Console.log in production (use structured logging)
- ❌ Hardcoded secrets (use env vars)
- ❌ Non-typed responses (always define types)
- ❌ Direct DB queries without error handling

## TypeScript Config
- `target: ES2020`
- `module: commonjs`
- `strict: true`
- `esModuleInterop: true`
